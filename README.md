# Разработка финансового виджета

## Краткое описание 
Проект является частью учебного процесса и предназначен для 
отработки практических навыков по программированию на Python
и работе с инструментарием разработчика (Pycharm, Poetry, GIT и т.п.)
Проект находится в стадии непрерывного развития. 

## Установка и использование
1. Клонируйте репозиторий:
```
git clone https://github.com/vicburnew/Home_work.git
```
2. Установите зависимости:
```
poetry install 
```
3. Запустите модуль main.py из корневой директории проекта 
```
python main.py
```



## Примеры использования 
В проекте реализовано несколько функций, 
расположенных в пакете .src. Вызов функций реализован из 
модуля main.py. Для тестирования модулей используется пакет pytest. 
Широко применяются фикстуры и параметризация.

7. filter_by_currency()

*Пример выхода от 1 до 5:*

*0000 0000 0000 0001

10. Декоратор @log  
    *Декоратор, который автоматически логирует начало и конец выполнения функции,
    а также ее результаты или возникшие ошибки. Принимает необязательный аргумент
    filename, который определяет, куда будут записываться логи (в файл или в консоль).
    Если filename задан, логи записываются в указанный файл.
    Если filename не задан, логи выводятся в консоль
    Логирование включает:  
    Имя функции и результат выполнения при успешной операции.  
    Имя функции, тип возникшей ошибки и входные параметры,
    если выполнение функции привело к ошибке.*  


11. Функция read_json_file()
    *Функция принимает на вход путь до JSON-файла и
    возвращает список словарей с данными о финансовых транзакциях.
    Если файл пустой, содержит не список или не найден,
    функция возвращает пустой список*   


12. Функция convert_currency_to_rub()  
    *Функция принимает на вход транзакцию и возвращает сумму транзакции
    (amount) в рублях, тип данных — float. Если транзакция была в USD или EUR,
    происходит обращение к внешнему API для получения текущего курса валют
    и конвертации суммы операции в рубли.
    Для конвертации валюты используется Exchange Rates Data API*   


13. Функция read_csv_file()

    функция возвращает пустой список.*

### Тестирование

В проекте написаны тестовые функции для контроля правильности кода
при различных значениях входных параметров с использованием фикстур и параметризации, а также "заглушек" Mock и patch. 
Результаты с отчетом покрытия тестами в формате HTML хранятся в репозитории проекта в папке /htmlcov.  
Для анализа покрытия кода нужно в любом браузере открыть файл index.html.  

### Конфиденциальные данные  

Для использования функции convert_currency_to_rub() требуется доступ к внешнему API. 
Вам необходимо пройти регистрацию на ресурсе https://apilayer.com/, получить персональный API-ключ и 
подставить его значение в файл .env, параметр API_KEY=, который вы создадите из шаблона .env.example, расположенного в 
корневом каталоге проекта. Файл .env должен быть размещен также в корневом каталоге проекта.
